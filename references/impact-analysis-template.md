# 改动影响分析模板

评估代码修改的影响范围和风险。

---

## 模块影响级别

| 模块            | 被依赖数 | 影响级别 | 修改风险       |
| --------------- | -------- | -------- | -------------- |
| `core/engine`   | 8        | 高       | ⚠️ 需要全面测试 |
| `shared/utils`  | 12       | 高       | ⚠️ 影响范围广   |
| `services/api`  | 4        | 中       | 需要集成测试   |
| `ui/components` | 2        | 低       | 局部影响       |

---

## 变更传播路径

### 高影响修改

- **修改 `shared/utils`** → 影响所有业务模块 → 全量回归测试
- **修改 `core/`** → 影响核心功能 → 需要完整测试套件

### 中影响修改

- **修改 `services/api`** → 影响前端 → E2E 测试
- **修改 `middleware/`** → 影响请求处理 → API 测试

### 低影响修改

- **修改 `ui/components`** → 局部 UI → 视觉测试
- **修改 `pages/`** → 单页面影响 → 页面测试

---

## 测试策略

| 修改范围 | 推荐测试            |
| -------- | ------------------- |
| 工具函数 | 单元测试            |
| 业务逻辑 | 单元测试 + 集成测试 |
| API 端点 | 集成测试 + E2E      |
| UI 组件  | 单元测试 + 视觉回归 |
| 全局配置 | 全量回归测试        |
